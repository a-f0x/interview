# ORD-42: Реализовать REST API управления доставкой

## Описание

Необходимо реализовать REST API для управления жизненным циклом доставки по существующим заказам. Каркас проекта
(модели, enum-ы статусов, репозиторий заказов, HTTP-клиент курьерской службы) уже подготовлен в main.

## Бизнес-требования

**Доставка:**

- Создание доставки для заказа (только если заказ в статусе `DELIVERY`; одна доставка на заказ)
- Продвижение доставки по статусной модели: `NEW` → `READY` → `IN_PROGRESS` → `FINISHED`
- При переходе в статус `READY` — отправка уведомления в службу доставки (POST
  `http://courier-service/api/notifications` с телом `{orderId, address}`)
- При переходе в статус `FINISHED` — автоматический перевод заказа в `FINISHED`

## Статусная модель

```
Заказ:    NEW → ASSEMBLY → DELIVERY → FINISHED
                                  ↘ CANCELLED (из NEW, ASSEMBLY)

Доставка: NEW → READY → IN_PROGRESS → FINISHED
```

> Управление статусами заказа — вне скоупа этой задачи; доставка только читает заказ и переводит его в `FINISHED` по
> завершении.

## API эндпоинты

| Метод | URL                            | Описание                                      |
|-------|--------------------------------|-----------------------------------------------|
| POST  | `/api/deliveries`              | Создать доставку (тело: `{orderId, address}`) |
| POST  | `/api/deliveries/{id}/advance` | Продвинуть статус доставки                    |

## Нефункциональные требования

- Операции изменения статуса должны быть атомарны и безопасны при конкурентном доступе
- Входные данные должны валидироваться
- API должен возвращать корректные HTTP-статусы

## Схема БД

Миграция Flyway. Таблицы: `orders`, `delivery` (FK на `orders`).
